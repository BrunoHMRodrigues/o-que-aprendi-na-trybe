Unix & Bash

Unix, Kernel e Shell

Unix: É um Sistema Operacional, usado como base para vários sistemas operacionais subsequentes e por isso considerado o pai dos sistemas operacionais. características: portável, multitarefas.

portável: Pode ser adaptado para os mais diversos sistemas computacionais facilmente

Kernel: É o núcleo do sistema operacional. Responsável por fazer a interação entre o Hardware e os Softwares. Ele gerencia os recursos do sistema, é quem faz a detecção de todo o hardware necessário para funcionamento da máquina quando o computador é ligado e é ele quem administra as principais funções da máquina, além do gerenciamento da memória, processos e arquivos do computador.

O Kernel traduz comandos do usuário em instruções para a máquina

Shell: É a interface entre o sistema operacional, usuário e o núcleo do sistema. Interface linha de comando e interface gráfica de usuário. Função: Ler linha de comando, interpretar, executar comando e devolver seu resultado ao usuário.

Exemplos de Shell: SH, BASH, CSH, TCSH, ZSH

Aplicativos: Softwares usados pelo usuário e maioria dos comandos do Unix

Processos: Tudo que é realizado no Unix: tarefas e comandos realizados. Processos possuem identificadores (PID) que carregam diversas informações sobre eles.

Arquivos: São os arquivos, dispositivos e até mesmo os diretórios.

Organizados em hierarquia de raiz, com seu nível mais alto sendo o "/" (diretório raiz). Arquivos tem uma série de informações atreladas a eles:

Arquivos tem uma série de informações atreladas a eles:

Permissões: Tipo de item e nível de permissão para leitura, escrita e execução do item ou diretório.

Links: Quantidade de ligações que o item possui

Proprietário: Dono do arquivo ou diretório, a pessoa que o criou

Grupo: O grupo ao qual o arquivo ou diretório pertence. Serve para dar permissões a outras pessoas

Tamanho: Tamanho em bytes

Data e Hora: Momento da ultima atualização, criação ou modificação

Nome: Nome do arquivo ou diretório

Git, Github e Internet

Git

Git é um sistema de versionamento. Ele possibilita que seja possível criar pontos de acesso para alterações realizadas de forma que você possa acessá-las quando for necessário. Dessa forma, caso ocorra algum problema com um programa é possível voltar para uma versão anterior do mesmo onde ele funcionava normalmente, isso sem mencionar a maior organização de um projeto que usa o Git, afinal não será mais necessário a criação de diversos arquivos de diferentes nomes com versões diferentes do projeto, sendo muito mais fácil de gerenciar projetos.

O versionamento do Git parte de um projeto principal, comumente conhecido como "main", os projetistas partem dele criando novas ramificações (branch) onde alterações serão realizadas para serem unidas a versão principal, dessa forma é possível imaginar que o versionamento de um projeto é como uma árvore com ramos que ao serem finalizados vão sendo unidos ao tronco que aos poucos se desenvolve.

Enquanto se desenvolve uma ramificação do projeto são salvos pontos de alterações (commit), esses pontos podem ser acessados caso necessário e mostram o desenvolvimento daquele ramo do projeto e quando o ramo é finalmente terminado é realizado a união (merge), ou mesclagem, dele ao ramo principal (branch main).

Repositórios

Repositório Local: Repositório no seu computador (teinha sido baixado da internet ou criado por você) Repositório Remoto: Repositório acessível na internet, hospedado em alguma plataforma de hospedagem como o Github.

Boas Práticas

  • Atualmente se costuma nomear a branch principal como main: Você pode usar o seguinte comando para configurar o nome main como padrão: git config --global init.defaultBranch main
  • Evite trabalhar no branch main: Lembre de trabalhar em ramificações para poder realizar alterações sem medo, afinal, ao trabalhar em uma ramificação você não estará afetando o projeto principal de forma a não gerar riscos ao projeto caso algum problema ocorra
  • Ao criar uma branch se certifique em que branch você se encontra: A branch é como um filho da branch atual, tenha cuidado para não criar uma branch a partir de uma errada
  • Realize commits com frequência: os commits ajudam a acompanhar as alterações realizadas, a correção de erros no código e funcionam como pontos de retorno caso algo dê errado.

Github

Plataforma é uma plataforma de hospedagem onde você pode salvar e gerenciar seus projetos de forma remota.

Autenticação: Para trabalhar com o Github é necessário criar uma conexão entre o Git (local) e o Github (remoto) e para isso é necessário fazer uma autenticação.

HTTPS ou SSH

HTTPS(Hypertext Transfer Protocol Secure): Extensão do protocolo de internet HTTP que utiliza certificados digitais para autenticar dados e permite que sejam criptografados.

Ao utilizar o HTTPS você precisará criar um token de acesso que será utilizado no lugar de sua senha quando for necessário.

SSH (Secure Shell): Protocolo ode criptografia que permitira transferir dados de forma segura sem que seja necessário digitar seu nome e chave de acesso pessoal ao executar cada comando.

Comunicação repositórios: remoto <---> Local

Ao se trabalhar com plataformas de hospedagem embora a forma de trabalhar em si seja a mesma:

- se cria uma branch onde as alterações serão realizadas

- faz-se os commits com frequência criando ponstos de salvamento e descrevendo as alterações que vão sendo realizadas

- se envia as alterações para o repositório remoto

A forma de conectar esses repositórios pode sofrer uma leve diferença de acordo por onde ela será criada: A partir de um repositório existente Local que deve ser conectado ao remoto, ou um remoto já existente que deverá ser trazido para um diretório local em sua máquina.

    Internet

    Internet: a rede de computadores que permite que nos comuniquemos por todo o mundo, enviando e recebendo dados

    Cliente: É o navegador: Firefox, Mozilla, Chrome, internet Explorer. O seu navegador está conectado a internet e é ele que recebe os seus comandos e o envia como um requisito para o servidor.

    Servidor: É um computador assim como o seu que também está conectado a internet. Ele tem o objetivo de conectar outros computadores em uma rede, o que demanda um alto poder de armazenamento e processamento. É o servidor que irá receber as requisoções enviadas pelo cliente e as irá processar, fornecendo os resultados necessários e o enviando de volta para o cliente.

    Endreço IP: É a identificação de computadores em uma rede, você pode imaginá-lo como o CPF de um computador. O IP é uma abreviação para Internet Protocol e segue um padrão de quatro campos separados por pontos, por exemplo: 211.122.48.2.

    TCP/IP: É um protocolo de envio e recebimento de dados. O Transmission Control Protocol (TCP) é quem define como os aplicativos se comunicam na rede e como os dados serão divididos em pacotes, já o IP como os pacotes deverão ser enviados para chegarem em seus destinatários. O TCP/IP então é a união dos dois protocoolos mencionados e pode ser dividida em 4 camadas: aplicação, transporte, rede e interface.

    ISP: Internet Service Provider (ISP) é o seu provedor de internet.

    DNS: Domain Name System (DNS) é um dispositivo com uma base de dados distribuída que gerencia os nomes de serviços, computadores ou qualquer dispositivo conectado a internet. Ele é usado para relacionar o nome endereço de sites que você está acostumado, como: www.google.com, com os seus endereços IP.

    Você sabia que um endereço como www.google.com.br ou www.w3schools.com na verdade é como uma máscara? Ali por trás, na verdade há um numero chamado IP, algo como: 192.168.5.5. Você consegue imagina se toda vez que você quisesse acessar algum site você tivesse que digitar um código como esse?

    Port Number: É um número de porta utilizado para identificar uma porta específica no servidor. Ela está relacionada ao endereço IP e é uma forma de identificar um processo específico encaminhado para o servidor.

    Host: É Qualquer dispositivo conectado à rede que possa transmitir ou receber pacotes IP. Exemplos: smartphones, tablets, computadores e muito mais.

    HTTP: Hyper-text Transfer Protocol (HTTP) é o protocolo utilizado para realizar a comunicação entre navegadores e os servidores na internet.

    URL: Uniform Resource Locators (URL) identifica um recurso web específico. Ele identifica o protocolo que será utilizado para nos comunicarmos com o servidor (https), o nome do host e o recurso que queremos acessar.

    Modelo Cliente-Servidor

    Você já se perguntou o que acontece quando você escreve um endereço de um site e aperta Enter? Tem ideia do que acontece ali, além da tela que você pode ver, para que o site finalmente apareça à sua frente?

    • Vocẽ interage com seu navegador (cliente) desejando entrar no site
    • O protocolo TCP/IP irá juntar os dados em pacotes e decidir como eles deverão ser enviados
    • O protocoo DNS fará a conversão do endereço nominal para o endereço IP
    • A requisição passa por um Balanceamento de carga: onde é traçado a melhor rota para um dos servidores otimizando tempo de resposta
    • O servidor cria um pacote de dados que é enviado para o usuário
    • Seu navegador recebe o pacote de dados e o processa exibindo para você a página
